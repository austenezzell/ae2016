Grid

Bruno requires a few classes. The .wrap class wraps all elements. It is horizontally centered and has a max-width. Columns are wrapped in a .row class, and each column requires a .col- class. Use the .col- classes to span the columns across different breakpoints. The default number of columns is 12. Max-width, column number, and gutter size can be customized in _config.scss.

Mobile first

Column widths start at mobile and scale to desktop. For example the "md" breakpoint will override the "xs" breakpoint on medium-sized screens.

Each tier of classes scales up from "xs", meaning if you plan on setting the same widths for "lg" and "md", you only need to specify "md". "lg", "md", and "sm" grids automatically stack at "xs" width.

Mixing breakpoints

The Uno grid system has four tiers of classes: xs (phones), sm (tablets), md (small desktop), and lg (full-width). You can combine these classes to create more flexible layouts.

.col-xs-12 .col-lg-8
.col-xs-6 .col-lg-4
.col-xs-4 .col-lg-5
.col-xs-4 .col-lg-2
.col-xs-4 .col-lg-5
.col-xs-6 col-sm-4 col-md-8 col-lg-6
.col-xs-6 col-sm-8 col-md-4 col-lg-6

Offset

Offset n-columns left and reset at specific breakpoints. Just subtract 1 column from the element you add the offset to, and be considerate of siblings. Offsets larger than "xs" automatically reset at "xs" width.

.col-sm-5 .col-xs-5 .offset-lg-1 .offset-sm-1
.col-sm-5 .col-xs-5 .offset-lg-1 .offset-sm-1
.col-sm-5 .col-xs-5 .offset-lg-1 .offset-sm-1
.col-sm-5 .col-xs-5 .offset-lg-1 .offset-sm-1
.col-xs-10 .offset-xs-2 .col-sm-12 .offset-sm-0 .col-md-6 .offset-md-6

Push / Pull

By default, all coulmns are floated left.However, on some breakpoints you may want an element on the the right and others on the left.

Using the .push and .pull class reorder you columns. Be advised, using this class can have adverse effects on the order of your elements, and weird stuff can happen if you're not careful.

[first in DOM] .col-xs-8 .push-lg-4
[second in DOM] .col-xs-4 .pull-lg-4

Nesting is possible and you can do some really weird stuff.

.col-xs-8 .push-lg-2
[nested and pulled] .col-xs-6 .pull-xs-1
[nested and pushed] .col-xs-6 .push-xs-1

Aligning Rows and Columns

Adding these classes to the .row element allows flexbox to work its magic.

.align-cols-top-, .align-cols-middle, and .align-cols-bottom- refer to the alignment of all columns inside the container.

.align-cols-left-, .align-cols-center-, and .align-cols-right- refer to the content inside the columns. It's a bit like text-align but works on blocks.

They can be mixed and matched in lieu of the push/pull classes to align content.

.top-xs .start-xs
.top-xs .start-xs
.middle-xs .center-xs
.middle-xs .center-xs
.bottom-xs .end-xs
.bottom-xs .end-xs

Adding .align-top-, .align-middle, and .align-bottom- to a .col element applies the alignment individually for more dynamic layouts. These override the alignment class on the parent, so be careful.

.top-xs
.middle-xs
.bottom-xs
